üë§ Setup Account Cloudflare

1Buat Account Cloudflare

Kunjungi cloudflare.com dan buat account baru:

  1. Klik tombol "Sign Up"
  2. Masukkan email dan password yang kuat
  3. Verifikasi email yang dikirim ke inbox Anda
  4. Login ke dashboard Cloudflare
üì∏

Dashboard Cloudflare akan tampil kosong karena belum ada domain yang ditambahkan

‚ûï Menambahkan Domain ke Cloudflare

2Add Site (Domain)

Tambahkan domain Anda ke Cloudflare:

  1. Di dashboard Cloudflare, klik "Add Site"
  2. Masukkan nama domain Anda (contoh: example.com)
  3. Klik "Add Site"
  4. Pilih plan "Free" (sudah sangat cukup untuk sebagian besar kebutuhan)
  5. Klik "Continue"
üí° Tips Pemilihan Plan

Plan Free Cloudflare sudah mencakup:

  • Bandwidth unlimited
  • SSL certificate gratis
  • DDoS protection dasar
  • CDN global
  • Analytics dasar

3Review DNS Records

Cloudflare akan melakukan scan otomatis untuk mendeteksi DNS records yang sudah ada:

  • Review records yang terdeteksi
  • Pastikan record A untuk domain utama menunjuk ke IP server Anda
  • Tambahkan record yang belum terdeteksi (akan dijelaskan di langkah berikutnya)
  • Klik "Continue"

‚ö†Ô∏è Proxy Status Explained

5Menambahkan DNS Record

Untuk menambah DNS record baru:

  1. Di dashboard domain, klik tab "DNS"
  2. Klik tombol "Add record"
  3. Pilih Type (A, AAAA, CNAME, MX, TXT, dll)
  4. Masukkan Name (subdomain atau @ untuk root domain)
  5. Masukkan Content (IP address atau target)
  6. Set Proxy status sesuai kebutuhan
  7. Klik "Save"

üéÆ Game Server Tips

Game servers harus menggunakan DNS Only karena:

  • Cloudflare proxy hanya mendukung HTTP/HTTPS traffic
  • Game servers menggunakan protokol TCP/UDP custom
  • Direct connection diperlukan untuk latency rendah

üîí SSL/TLS Configuration

9Aktifkan SSL/TLS

Setup SSL certificate gratis dari Cloudflare:

  1. Di dashboard domain, klik tab "SSL/TLS"
  2. Di bagian "Overview", pastikan SSL/TLS encryption mode diset ke "Full (strict)"
  3. Klik tab "Edge Certificates"
  4. Pastikan "Always Use HTTPS" aktif (ON)
  5. Aktifkan "HTTP Strict Transport Security (HSTS)"
SSL Mode Security Level Keterangan
Off ‚ùå None Tidak ada enkripsi
Flexible üîí Low Cloudflare ke visitor terenkripsi
Full üîíüîí Medium End-to-end encryption
Full (strict) üîíüîíüîí High Valid SSL certificate required
üéâ Rekomendasi

Gunakan "Full (strict)" untuk keamanan maksimal. Pastikan server Anda memiliki SSL certificate valid (bisa dari Let's Encrypt).

10Setup Origin Certificates

Untuk mode "Full (strict)", buat Origin Certificate:

  1. Di tab "Origin Server"
  2. Klik "Create Certificate"
  3. Pilih "Let Cloudflare generate a private key and a CSR"
  4. Masukkan domain dan subdomain (contoh: *.example.com, example.com)
  5. Pilih validity "15 years"
  6. Klik "Create"
  7. Download dan install certificate di server Anda

‚ö° Speed Optimization

11Caching Configuration

Optimasi caching untuk performa maksimal:

  1. Klik tab "Caching"
  2. Di "Configuration":
    • Caching Level: "Standard"
    • Browser Cache TTL: "4 hours"
    • Always Online‚Ñ¢: "On"
  3. Di "Cache Rules", buat rules untuk static assets:
    • Match: *.css, *.js, *.png, *.jpg, *.jpeg, *.gif, *.ico
    • Cache Everything
    • Edge TTL: "1 month"

12Speed Optimizations

Aktifkan fitur speed optimization:

  1. Klik tab "Speed"
  2. Di "Optimization":
    • Auto Minify: Aktifkan untuk CSS, JavaScript, HTML
    • Brotli: On (kompresi yang lebih efisien dari gzip)
    • Early Hints: On (HTTP/2 push optimization)
  3. Di "Polish" (untuk Pro plan):
    • Image optimization otomatis
    • WebP conversion

üîß Troubleshooting

üö® Common Issues

Masalah umum dan solusinya:

‚ùå Website Tidak Bisa Diakses

Kemungkinan Penyebab:

  • DNS belum propagasi (tunggu 24-48 jam)
  • DNS record salah (cek A record menunjuk ke IP yang benar)
  • Server down (cek status server)

Solusi:

  1. Cek DNS propagation di whatsmydns.net
  2. Verifikasi A record di Cloudflare DNS tab
  3. Test direct IP access (bypass Cloudflare)
  4. Cek server logs untuk error

üîí SSL Certificate Error

Error "Your connection is not private":

  • SSL mode tidak sesuai dengan konfigurasi server
  • Origin certificate belum diinstall
  • Mixed content (HTTP resources di HTTPS page)

Solusi:

  1. Pastikan SSL mode di Cloudflare sesuai:
    • Flexible: Jika server tidak punya SSL
    • Full: Jika server punya self-signed SSL
    • Full (strict): Jika server punya valid SSL
  2. Install Origin Certificate dari Cloudflare
  3. Fix mixed content warnings

üêå Website Lambat

Optimasi yang Bisa Dilakukan:

  1. Pastikan caching rules aktif
  2. Aktifkan Brotli compression
  3. Enable Auto Minify untuk CSS/JS
  4. Optimasi images di server
  5. Review Page Rules untuk bypass cache pada dynamic content

üö´ 502/504 Bad Gateway Error

Penyebab dan Solusi:

  • Server overload: Scale up server resources
  • Wrong origin server: Cek A record menunjuk ke IP yang benar
  • Firewall blocking Cloudflare: Whitelist Cloudflare IP ranges
  • Server software issue: Restart web server (nginx/apache)
